When I type google.com and press Enter, my browser first checks the Domain Name System (DNS) to find the IP address of the website. After getting the IP address, it connects to the web server over the internet. The server processes the request and sends back the webpage files, which the browser displays on my screen.
A client is a device that requests information or services from another computer and a server is a computer or program that provides those services.Example of a client: Google Chrome web browser.
Example of a server: Google's web server that hosts google.com.
DNS (Domain Name System) is a system that translates human-readable domain names, such as google.com, into IP addresses that computers use to communicate. We need DNS because it is much easier to remember website names than numerical IP addresses
A firewall protects a network by allowing or blocking incoming and outgoing traffic based on security rules. It helps prevent unauthorized access and cyber attacks. A load balancer distributes incoming traffic across multiple servers so that no single server becomes overloaded. This improves performance and reliability. The main difference is that a firewall focuses on security, while a load balancer focuses on performance, availability, and efficient use of server resources.
A web server mainly serves static files such as HTML, CSS, JavaScript, and images to users. An application server runs backend code, processes business logic, communicates with databases, and generates dynamic content. A backend developer works more with the application server because it handles the application's logic, user authentication, data processing, and communication with the database.
Clients do not communicate directly with the database because it would expose sensitive data and create security risks. Instead, requests go through an application server.
The application server: Authenticates and authorizes users, Validates user input, Protects the database from unauthorized access, Applies business rules.
A university department page is more likely to be a static website because its content does not change frequently and is generally the same for all visitors. An MTN MoMo account page is a dynamic website because it displays personalized information such as account balance, transaction history, and user details after logging in. The content changes depending on the user and the database.

Backend development focuses on the server, application logic, and database operations that happen behind the scenes. For a login feature, the backend is responsible for checking the user's password against the database because this process involves sensitive information and must be handled securely. The frontend only collects the user's login details and sends them securely to the backend for verification.
Backend developers need to know how to use the command line because it allows them to work more efficiently when creating, managing, and deploying applications. Many development tools, servers, databases, and version control systems such as Git are primarily used through the command line. It also makes it easier to automate tasks and work on remote servers.
An absolute path is the complete path to a file or folder starting from the root directory. It always points to the same location regardless of the current directory. Example:/home/ruth/t4g_assignment/class01/notes.txt A relative path is a path that is based on the current working directory. It depends on where you are currently located. Example:class01/notes.txt
I would explain that File Explorer and Finder are useful for basic file management, but the command line is faster and more powerful for many tasks. It allows you to automate repetitive work, manage files efficiently, use development tools, access remote servers, and perform tasks that are difficult or impossible with a graphical interface. Since many backend development tools rely on the command line, learning it is an important skill.
The ~ symbol represents the user’s home directory. It is a shortcut that allows you to quickly navigate to your home folder. The $ symbol is the shell prompt for a regular user. It indicates that the terminal is ready to accept a command. On Linux systems, the # symbol is usually used for the root (administrator) user instead.

========================================================================================

 (Browser/User) <---------------( 2. Returns IP )--------------------- (Domain Name System)
     |
     | (3. Sends HTTP Request to IP)
     v
 [ FIREWALL ] --------------------------------------------------------> (Monitors & blocks unauthorized traffic)
     |
     | (4. Clean traffic passes through)
     v
 [ LOAD BALANCER ] ---------------------------------------------------> (Distributes incoming requests across servers)
     |
     | (5. Routes to available server)
     v
 [ WEB SERVER ] ------------------------------------------------------> (Handles HTTP requests, serves static files)
     |
     | (6. Forwards dynamic request)
     v
 [ APPLICATION SERVER ] ----------------------------------------------> (Runs business logic and processes data)
     |
     | (7. Queries data)
     v
 [ DATABASE ] <======================================================== (Stores and manages application data)

                      THE RETURN JOURNEY (SENDING THE RESPONSE)
========================================================================================

 [ DATABASE ] ------------------( 8. Sends data back )----------------> [ APPLICATION SERVER ]
                                                                                |

     |
     | (10. Passes response back)
     v
 [ LOAD BALANCER ] ---> [ FIREWALL ] ---> [ CLIENT ] (11. Browser renders the page for the user!)
